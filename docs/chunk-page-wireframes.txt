================================================================================
                              COLUMN DEFINITIONS
================================================================================

For use with FilterableTable component:

Column ID    │ Title       │ Filter Type   │ Notes
─────────────┼─────────────┼───────────────┼─────────────────────────────────────
checkbox     │ (none)      │ none          │ Row selection
location     │ Location    │ multiselect   │ Dynamic options from data
dateTime     │ Date/Time   │ multiselect   │ Format: "Oct 25, 9:00 AM"
matchup      │ Matchup     │ search        │ "Home vs Away" + Division below
division     │ Division    │ multiselect   │ Could be hidden, shown in matchup
level        │ Level       │ multiselect   │ Rec/Comp/Elite badges
refs         │ Refs        │ none          │ "assigned/required" format
status       │ Status      │ multiselect   │ Color dot indicator



================================================================================
                                RECOMMENDATION
================================================================================

Option 1 (Flat Table with Filters) is recommended because:

1. Uses existing FilterableTable component - minimal new code
2. Eliminates nested accordions completely
3. Multi-select filters give same drill-down capability
4. Easy to scan all games at once
5. Familiar table pattern users already know
6. Mobile card view already supported by component

The grouped Option 2 is nice for visual organization but adds complexity.
Option 3 is most compact but may feel cramped.
